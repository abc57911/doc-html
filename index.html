工作流程 Skills

6 個
writing-plans Skill

當你有規格或需求，但尚未觸及程式碼時，使用此 skill 來撰寫全面的實作計畫。必須在專用的 worktree 中執行。

writing planning
executing-plans Skill

當已有現成的實作計畫時使用，在獨立 session 中執行並設置審查 checkpoint。核心原則是「批次執行搭配架構審查」。

execution checkpoints
subagent-driven-development Skill

透過分派獨立的子代理來執行實作計畫，每個任務後進行兩階段審查：先檢查規格符合度，再檢查程式碼品質。

subagent review
finishing-a-development-branch Skill

當實作完成、測試通過後，用於指導完成開發工作的流程。提供 Merge、PR、保持現狀、丟棄四個選項。

git workflow
test-driven-development Skill

實作任何功能或錯誤修正前，先撰寫測試。看著它失敗，然後寫最少量的程式碼讓它通過。核心原則：Red-Green-Refactor。

testing TDD
planning-with-files Skill

使用 Manus 風格的持久化 Markdown 檔案進行規劃、進度追蹤和知識儲存。適用於複雜任務、多步驟專案。

files tracking

    • 核心 Tools

    9 個

    核心 Tools 是所有 agents 都可以使用的基本工具，用於檔案操作、命令執行和程式碼搜尋。

    Read

    ▼

    讀取檔案內容。可以指定偏移量和限制行數，適用於檢視大型檔案的特定部分。

  • 用途：查看檔案內容、閱讀程式碼
  • 參數：file_path, limit, offset
  • 範例：讀取前 100 行 `Read file_path: "/path/to/file" limit: 100`

    • Write

    ▼

    建立新檔案或覆寫現有檔案。如果檔案已存在，必須先使用 Read 工具讀取。

  • 用途：建立新檔案、寫入內容
  • 參數：file_path, content
  • 範例：`Write file_path: "/new/file.txt" content: "Hello World"`

    • Edit

    ▼

    對現有檔案進行精確的字串替換。必須確保 old_string 在檔案中唯一。

  • 用途：修改程式碼、編輯內容
  • 參數：file_path, old_string, new_string
  • 範例：`Edit file_path: "/file.js" old_string: "old text" new_string: "new text"`

    • Bash

    ▼

    執行 Shell 命令。可用於 Git 操作、套件管理、建置流程等。支援背景執行和超時設定。

  • 用途：執行命令、Git 操作、安裝依賴
  • 參數：command, description, timeout, run_in_background
  • 範例：`Bash command: "npm install" description: "Install npm packages"`

    • Glob

    ▼

    使用 glob 模式搜尋檔案。支援萬用字元如 `**/*.js`、`src/**/*.ts` 等。

  • 用途：搜尋符合模式的檔案
  • 參數：path, pattern
  • 範例：`Glob pattern: "**/*.test.js"`

    • Grep

    ▼

    使用正規表示法搜尋檔案內容。可以限制檔案類型、顯示行號、取得計數等。

  • 用途：在程式碼中搜尋關鍵字、函式、變數
  • 參數：pattern, glob, output_mode, head_limit
  • 範例：`Grep pattern: "function.*test" glob: "**/*.js"`

    • Task

    ▼

    派遣 subagent 執行任務。可以指定 agent 類型、超時時間、最大回合數。

  • 用途：派遣專門的 agent 處理特定任務
  • 參數：subagent_type, prompt, max_turns, model
  • 範例：`Task subagent_type: "code-reviewer" prompt: "Review this PR..."`

    • WebFetch

    ▼

    從 URL 擷取內容並使用 AI 模型處理。自動將 HTML 轉換為 Markdown。

  • 用途：擷取網頁內容、分析文件
  • 參數：url, prompt
  • 範例：`WebFetch url: "https://example.com" prompt: "Extract main content"`

    • WebSearch

    ▼

    搜尋網頁並取得結果。支援分頁、過濾、新聞搜尋等功能。

  • 用途：網頁搜尋、取得最新資訊
  • 參數：query, count, offset
  • 範例：`WebSearch query: "latest React documentation 2024" count: 10`
